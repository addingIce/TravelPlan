# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This repo stores travel plan HTML files generated by the `travel-plan-viz` skill (`/travel-plan-viz`). Each file is a self-contained, single-page travel plan with inline JavaScript, embedded trip data (`#trip-data` JSON), and styled itinerary.

## 必须调用技能（每次生成/修改时优先于静态调研）

- **`/amap-lbs-skill`** — 坐标 (GCJ-02)、自驾路线规划、导航链接。地图统一用高德（Amap iframe），**不用 Leaflet/OSM**。
- **`/flyai`** — `search-hotel` / `ai-search` / `keyword-search` 拿酒店实时价+预订链接；`search-poi` 拿景点图片 (`mainPic`)。

## 基于模板

所有新旅行计划 **基于 `模板-旅行计划.html` 复制**，填入具体内容即可。模板已包含完整 CSS/JS/结构，避免从头搭建。

## UI 规范（照搬金华文件，即模板样式）

### 页面区块顺序
1. **Hero 封面** — 渐变深色全宽，含行程名、日期、统计数字
2. **待办清单** — 浮在 hero 下方的 `panel` 卡片
3. **行前须知** — `notice` 天气卡片 + `panel` 内 `dl > dt/dd` 列表
4. **交通方式** — `drive-card`（自驾默认）
5. **酒店** — `.hotel-area > .hotel-options > .hotel-option`（飞猪数据，含 `tier` 档次标签+预订链接）
6. **免责声明** — `.notice`（珊瑚左边框）
7. **高德行程地图** — Amap iframe 嵌入 + "在高德打开"链接
8. **每日时间轴** — `.day` > `.day-head`（`.day-kicker` + `.day-title` + `.day-tips` 内的 `.mini`） > `.slots`
9. **全程贴士** — `.tip-card` 列表
10. **数据来源** 标注

### 卡片结构 (`<article class="slot">`)
- `.photo` — `aspect-ratio: 16/9`, `min-height: 170px`，`object-fit: cover` + 渐变底色 + `::after` 显示 `data-name`
- `.slot-body` — 信息区，flex 列：
  - `.slot-top`：标题 h4 + 时间 .time + 评分 .rating | `.period` 徽章（右上角：🌅 上午 / ☀️ 中午 / 🌆 下午–傍晚）
  - `.review` 点评
  - `.meta > .pill` 药丸标签（开放/票价/交通/时令）
  - `.slot-route-info`：🚗 从上一站到此：X分钟 + 查看路线→
  - `.nav-actions`：两个胶囊按钮 `.nav-link--amap`（高德导航）和 `.nav-link--system`（系统地图）
- 桌面端 grid 35%/65%，手机单列

### 餐饮整合
- 午餐/晚餐卡片按时段自动插入对应 slot 组之后，`border-color:var(--warm)` 区分
- 全量配图（餐厅/补给点也尽量配图）

### Hero 封面
- `<div class="hero">` + `.hero-inner` + `.eyebrow` + `h1` + `.hero-stats > .stat`

## 技术要求

- **图片**: 首选 `flyai search-poi` 的 `mainPic`（alicdn.com）。URL 必须验证 200。`onerror="this.style.display='none'"`（单引号内嵌，不可用嵌套双引号）。
- **CSS**: 复制金华完整样式（`:root` 变量、卡片、按钮、面板结构）。配色根据目的地调整（如杭州茶绿、金华自然棕）。
- **HTML 结构**: 所有标签必须平衡（`<style>` 必须有 `</style>`，`<article>` 数量一致等）。
- **天气卡片**: 包含行程日期逐日预报（日间卡片，显示气温+天气+风力）。
- **校验**: 生成后必须跑 `validate.js`，ERROR 必修复。
- **`trip-data` JSON**: 所有结构化数据存于此，后续迭代改 JSON 重渲染，不反解 DOM。

## 已知陷阱

| 问题 | 表现 | 解决方法 |
|------|------|----------|
| `</style>` 缺失 | 白屏（浏览器把 head 当 CSS） | 替换 CSS 后检查 `</style>` 存在 |
| `onerror` 嵌套双引号 | 白屏（HTML 解析断裂） | `onerror="this.style.display='none'"` |
| 金华 CSS 的暖色变量 | 杭州文件黄屏 | 改 `:root` 变量为目的地配色 |
| slot 名与 JSON 不一致 | 导航按钮/图片匹配不上 | 从 trip-data JSON 精确取图，不靠全局替换 |
